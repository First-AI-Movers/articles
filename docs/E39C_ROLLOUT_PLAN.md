## 6. Review-gated workflow and exact commands

The workflow mirrors E35 (`build_summaries.py`) and E39b exactly. No automation calls DeepL without `--allow-network`. No `article.<lang>.md` is created without `--apply-approved`.

### Step 1 — Dry-run preview (no writes, no network)

Preview a single article in one language:

```bash
python3 tools/translate_articles.py \
  --dry-run \
  --slug claude-prompt-architecture-vs-complexity-2026 \
  --lang es \
  --provider mock
```

Preview an entire batch (character budget only):

```bash
python3 tools/translate_articles.py \
  --dry-run \
  --slug claude-prompt-architecture-vs-complexity-2026 \
  --lang es,fr,de,nl,pt \
  --provider mock
```

### Step 2 — Live quota check (no writes)

Verify DeepL Free quota before spending:

```bash
export DEEPL_API_KEY="<key>"
python3 tools/translate_articles.py \
  --dry-run \
  --slug claude-prompt-architecture-vs-complexity-2026 \
  --lang es,fr,de,nl,pt \
  --provider deepl \
  --allow-network
```

Expected output: `Quota check: PASSED (dry-run)` with remaining chars > projected chars + 5,000 margin.

### Step 3 — Generate review files (network required)

Generate draft review files for one article, all 5 languages:

```bash
python3 tools/translate_articles.py \
  --write-review-files \
  --slug claude-prompt-architecture-vs-complexity-2026 \
  --lang es,fr,de,nl,pt \
  --provider deepl \
  --allow-network \
  --translations-dir translations/reviews
```

Output: `translations/reviews/<slug>.<lang>.review.md` for each language.

**Do not commit review files until they have been human-reviewed.**

### Step 4 — Human review

For each review file:
1. Open `translations/reviews/<slug>.<lang>.review.md`.
2. Verify the **Terminology check** table against `tools/translation_glossary.json`.
3. Edit the translated body for tone, nuance, and brand voice.
4. Fill in:
   - `Status: approved`
   - `Reviewer: Dr. Hernani Costa`
   - `Reviewed at: YYYY-MM-DD`
5. Save the review file.

### Step 5 — Apply approved translations

After all 5 languages for an article are approved:

```bash
python3 tools/translate_articles.py \
  --apply-approved \
  --slug claude-prompt-architecture-vs-complexity-2026 \
  --lang es,fr,de,nl,pt
```

This creates:
- `articles/<folder>/article.<lang>.md`
- `articles/<folder>/translations.json` (updated with `status: published`)

### Step 6 — Validate before PR

```bash
python3 tools/check_translations.py
python3 -m pytest tools/tests/test_translate_articles.py -v
python3 -m pytest tools/tests/test_check_translations.py -v
python3 -m pytest tools/tests/test_multilingual_pages.py -v
python3 tools/rebuild_local.py
```

Verify `site/<lang>/articles/<slug>/index.html` renders correctly with:
- Self-canonical URL
- `index, follow` robots tag
- Dynamic `<html lang="...">`
- Dynamic JSON-LD `inLanguage`
- hreflang alternate links

### Step 7 — Open PR

PR title: `feat(translations): batch N — <slug1>, <slug2>, ...`

Commit each article's translations as a separate commit within the PR (atomic per article):

```
feat(translations): publish ES/FR/DE/NL/PT for <slug>
```

## 7. Risks and rollback

| Risk | Likelihood | Impact | Mitigation | Owner decision needed? |
|---|---|---|---|---|
| DeepL Free 500K chars/month is exceeded mid-batch | Low | High | Dry-run quota check before every `--write-review-files`; split batch if quota tight; upgrade to DeepL Pro ($5.49/mo + $25/M chars) if budget allows. | No — operational |
| Machine translation quality damages brand without review | Low | High | Review-gated workflow: `--apply-approved` refuses draft status; no CI automation calls DeepL. | No — workflow enforces |
| Terminology inconsistency across languages | Medium | Medium | `tools/translation_glossary.json` + terminology check table in every review file + post-apply `check_translations.py`. | No — tooling enforces |
| GoatCounter data becomes available and contradicts citation-graph ranking | Medium | Low | The candidate list can be reordered before any batch starts; already-translated articles are sunk cost. Replan is cheap. | No — rebalance before next batch |
| Build time increase from 911 → ~1,000+ pages | Low | Low | Measure during batch 1; `rebuild_local.py` is I/O-bound and scales linearly. Parallelize article rendering if needed. | No — measure first |
| hreflang cluster is technically imperfect (English = noindex, translations = index) | Certain | Low | Documented in `docs/E39_TRANSLATION_PLAN.md` Section 7 as Option B. Accepted pragmatic intermediate. Architect for Option C (full self-canonical cluster) after WordPress/Hetzner migration. | **YES** — Dr. Costa already approved Option B for pilot; re-confirm for E39c. |
| City/location-specific articles creep into future batches | Low | Medium | Explicit exclusion rule in Section 2; re-evaluate only if GoatCounter proves demand. | No — policy gates this |
| DeepL API rate-limit or outage during batch generation | Low | Medium | Tool has 3 retries with exponential backoff (1s, 2s, 4s). Batch can be resumed by targeting remaining slugs. | No — handled by tool |
| Reviewer bandwidth bottleneck (5 languages × 19 articles = 95 review files) | Medium | High | Batch quarterly if needed; Spanish + Portuguese share review effort (Iberian overlap); Dutch reviewer can be external. | **YES** — confirm reviewer availability before Batch 1. |

**Rollback per article:**
1. Edit `articles/<folder>/translations.json` and change language `status` from `"published"` to `"draft"`.
2. Run `python3 tools/rebuild_local.py`.
3. Translated HTML pages are no longer rendered; `article.<lang>.md` and review files remain for future re-approval.

**Rollback per batch:**
- Revert the PR merge commit. `article.<lang>.md` and `translations.json` files are removed. Review files in `translations/reviews/` are preserved if not committed (they should be committed for audit trail).
